# evidence-discipline.md — the fail-closed rule for claims and statistics

Source: [LinkedIn Content Architecture](https://app.notion.com/p/3a297ad57299817ab638fdd08d68d565) (Notion)

Drawn from the MWA 30-day campaign's two audit passes (fact-checking + brand/compliance), generalized — this file does not repeat or re-litigate those audits' specific dated verdicts.

## The fail-closed principle

**A fixed publish date never overrides a missing proof requirement.** Before publishing any post with a participant story, reward/payout detail, measured outcome, or statistic, check:

- **Consent** — documented consent/attribution record?
- **Dated evidence** — a specific artifact dated at or before publication?
- **Scope and denominator** — population, time window, what's actually measured?
- **Source verification** — checked against the primary source, not a secondhand summary?

If any is missing, the post is **held**. Always have an evergreen alternative ready (a current guide walkthrough, a sourced media-literacy lesson, a founder-owned field note with no participant claim).

## The three-part verdict framework

- **Verified** — publishable from cited evidence, subject to a normal day-of check.
- **Qualified** — core idea sound, but exact wording/source needs a specific correction.
- **Blocked / Hold** — a required evidence artifact doesn't exist yet.

## The claims ledger

Sort every claim about a feature into three tiers:

- **Safe now** — supportable directly from how the feature currently works.
- **Requires measurement** — true only if a comparison/sample/metric existed, and it doesn't yet. Don't state as fact.
- **Requires new functionality** — the product doesn't do this yet. Never state as current capability.

## The positioning ladder

Build a claim in layers: Category → Feature → Function → Practical value → Experience value → Relationship value → Brand idea. Same instinct as Ground Then Elevate, worked out more granularly.

## Sourcing discipline

Verify statistics against the primary, authoritative source. The repo's `research/content-calendar/09-source-pack.md` and its two audit files are a resource, not a shortcut — any reused figure still needs a fresh Verified/Qualified/Blocked check, since it was verified for a different platform's dated campaign.
